# Online Multiplayer Design for Shift Tac Toe

## How WebSockets Work (Primer)

Before diving in: a WebSocket is a persistent, two-way connection between a browser and a server. Unlike a normal HTTP request (which is "ask, get answer, disconnect"), a WebSocket stays open so either side can send a message at any time. This is exactly what a game needs — when your opponent makes a move, the server can push it to you instantly without you polling.

The flow for this game:

```
Browser A                Server               Browser B
   |                       |                      |
   |------ connect ------->|                      |
   |                       |<------- connect ------|
   |--CREATE_ROOM--------->|                      |
   |<--ROOM_CREATED(code)--|                      |
   |                       |<----JOIN_ROOM(code)---|
   |<--OPPONENT_JOINED-----|--GAME_START---------->|
   |                       |                      |
   |--PLAYER_ACTION------->|                      |
   |<--(echo it back)------|--PLAYER_ACTION------->|
   |                       |                      |
```

The server is purely a relay: it never runs `gameReducer`. Both clients run it independently. Because the reducer is deterministic, they always produce identical state from identical action sequences.

### `server/rooms.ts` — What it does

Maintains a `Map<string, Room>` where a `Room` is:

```typescript
interface Room {
  code: string;
  redSocket: WebSocket;       // always the creator
  blackSocket: WebSocket | null; // null until someone joins
}
```

Exports:
- `createRoom(socket)` — generates a 6-char code, stores the room, returns code
- `joinRoom(code, socket)` — looks up room, sets `blackSocket`, returns the room or null
- `getRoomForSocket(socket)` — reverse-lookup (needed for disconnect handling)
- `removeRoom(code)` — clean up when someone disconnects

Room code generation: `Math.random().toString(36).slice(2,8).toUpperCase()`. Simple, good enough.

### `server/index.ts` — What it does

```typescript
import { WebSocketServer, WebSocket } from 'ws';
import { createRoom, joinRoom, getRoomForSocket, removeRoom } from './rooms.js';

const wss = new WebSocketServer({ port: 8080 });

wss.on('connection', (socket) => {

  socket.on('message', (raw) => {
    const msg = JSON.parse(raw.toString()); // ClientMessage

    if (msg.type === 'CREATE_ROOM') {
      const code = createRoom(socket);
      send(socket, { type: 'ROOM_CREATED', roomCode: code, yourColor: 'red' });
    }

    else if (msg.type === 'JOIN_ROOM') {
      const room = joinRoom(msg.roomCode, socket);
      if (!room) {
        send(socket, { type: 'ERROR', reason: 'ROOM_NOT_FOUND' });
        return;
      }
      send(room.redSocket, { type: 'OPPONENT_JOINED' });
      send(socket,          { type: 'GAME_START', roomCode: room.code, yourColor: 'black' });
    }

    else if (msg.type === 'PLAYER_ACTION') {
      const room = getRoomForSocket(socket);
      if (!room) return;
      const opponent = socket === room.redSocket ? room.blackSocket : room.redSocket;
      if (opponent) send(opponent, { type: 'PLAYER_ACTION', action: msg.action });
    }
  });

  socket.on('close', () => {
    const room = getRoomForSocket(socket);
    if (!room) return;
    const opponent = socket === room.redSocket ? room.blackSocket : room.redSocket;
    if (opponent) send(opponent, { type: 'OPPONENT_LEFT' });
    removeRoom(room.code);
  });
});

function send(socket: WebSocket, msg: ServerMessage) {
  if (socket.readyState === WebSocket.OPEN) {
    socket.send(JSON.stringify(msg));
  }
}
```

This is the complete server logic — approximately 50 lines. Its only job is: receive a message from socket X, forward it to socket Y.

```typescript
  .slice(base.length)          // strip base path
  .match(/^game\/([A-Z0-9]{6})$/)?.[1] ?? null;

// Generate shareable URL
const shareUrl = `${window.location.origin}${base}game/${roomCode}`;
```

# Plan: Online Multiplayer for Shift Tac Toe

## Context

The game currently supports 1P (vs AI) and 2P (local). The goal is to add an `'online'` mode so two players on different browsers can play against each other. Joining is URL-based — Player A creates a room and gets a link like `/game/XKCD42`; Player B clicks it and auto-joins. No manual code entry.

The key insight that makes this straightforward: `gameReducer` is a **pure deterministic function**. Given the same sequence of actions, two clients will always produce identical state. So the server never needs to run game logic — it just relays actions from one player to the other. Both clients dispatch actions locally and stay in sync automatically.

## Architecture: Action Relay over WebSocket

```
Player A                   Server (Node.js)              Player B
  |--CREATE_ROOM----------->|
  |<--ROOM_CREATED("XKCD42")-|
  |  URL → /game/XKCD42      |
  |  (waiting for opponent)  |
  |                          |<--JOIN_ROOM("XKCD42")--------|
  |                          |  (server flips a coin)       |
  |<--GAME_START(red)--------|--GAME_START(black)---------->|  (or vice versa, random)
  |                          |                              |
  |  user clicks col 1       |                              |
  |  dispatch(DROP_DISC)     |                              |
  |--PLAYER_ACTION---------->|                              |
  |                          |--PLAYER_ACTION-------------->|
  |                          |                  dispatch(DROP_DISC)
```

**Color assignment:** when the joiner arrives, the server randomly picks who gets red and who gets black, and sends `GAME_START { yourColor }` to **both** players simultaneously. Neither player knows their color until the game actually starts. Red goes first (unchanged from existing game logic).

The server is ~50 lines. No game logic. Rooms live in memory and die on disconnect (intentionally simple).

# Plan: Online Multiplayer for Shift Tac Toe

## Context

The game currently supports 1P (vs AI) and 2P (local). The goal is to add an `'online'` mode so two players on different browsers can play against each other. Joining is URL-based — Player A creates a room and gets a link like `/game/XKCD42`; Player B clicks it and auto-joins. No manual code entry.

The key insight that makes this straightforward: `gameReducer` is a **pure deterministic function**. Given the same sequence of actions, two clients will always produce identical state. So the server never needs to run game logic — it just relays actions from one player to the other. Both clients dispatch actions locally and stay in sync automatically.

## Architecture: Action Relay over WebSocket

```
Player A                   Server (Node.js)              Player B
  |--CREATE_ROOM----------->|
  |<--ROOM_CREATED("XKCD42")-|
  |  URL → /game/XKCD42      |
  |  (waiting for opponent)  |
  |                          |<--JOIN_ROOM("XKCD42")--------|
  |                          |  (server flips a coin)       |
  |<--GAME_START(red)--------|--GAME_START(black)---------->|  (or vice versa, random)
  |                          |                              |
  |  user clicks col 1       |                              |
  |  dispatch(DROP_DISC)     |                              |
  |--PLAYER_ACTION---------->|                              |
  |                          |--PLAYER_ACTION-------------->|
  |                          |                  dispatch(DROP_DISC)
```

**Color assignment:** when the joiner arrives, the server randomly picks who gets red and who gets black, and sends `GAME_START { yourColor }` to **both** players simultaneously. Neither player knows their color until the game actually starts. Red goes first (unchanged from existing game logic).

The server is ~50 lines. No game logic. Rooms live in memory and die on disconnect (intentionally simple).

## URL Routing (no React Router needed)

The app has two "pages": lobby and game. We handle this with `window.location` and `history.pushState` directly.

**URL patterns:**
- `/` (or `/shift-tac-toe/` in production) → lobby / home
- `/game/XKCD42` (or `/shift-tac-toe/game/XKCD42`) → join/play that room

**On app load:** check `window.location.pathname` for the `/game/CODE` pattern. If found, auto-join that room immediately. If not, show the lobby.

**When creating a room:** after `ROOM_CREATED`, call `history.pushState` to update the URL and display the shareable link.

**URL generation uses `import.meta.env.BASE_URL`** to handle both dev (`/`) and production (`/shift-tac-toe/`):
```typescript
const base = import.meta.env.BASE_URL;
const roomCode = window.location.pathname
  .slice(base.length)
  .match(/^game\/([A-Z0-9]{6})$/)?.[1] ?? null;

const shareUrl = `${window.location.origin}${base}game/${roomCode}`;
```

**Browser back button:** listen for `popstate` and reset to lobby.

**nginx and Vite SPA fallback:** both already serve `index.html` for all paths, so deep links work without server-side routing changes.

import type { WebSocket } from 'ws';

export interface Room {
  code: string;
  redSocket: WebSocket | null;
  blackSocket: WebSocket | null;
}

const rooms = new Map<string, Room>();

// Reverse index so we can find a room given just a socket
const socketToRoom = new Map<WebSocket, Room>();

function generateCode(): string {
  // 6-char alphanumeric, uppercase
  return Math.random().toString(36).slice(2, 8).toUpperCase();
}

export function createRoom(socket: WebSocket): string {
  let code = generateCode();
  // Extremely unlikely collision, but be safe
  while (rooms.has(code)) code = generateCode();

  const room: Room = { code, redSocket: socket, blackSocket: null };
  rooms.set(code, room);
  socketToRoom.set(socket, room);
  return code;
}

export function joinRoom(code: string, socket: WebSocket): Room | null {
  const room = rooms.get(code);
  if (!room) return null;
  if (room.blackSocket !== null) return null; // already full

  room.blackSocket = socket;
  socketToRoom.set(socket, room);
  return room;
}

export function getRoomForSocket(socket: WebSocket): Room | null {
  return socketToRoom.get(socket) ?? null;
}

export function removeRoom(code: string): void {
  const room = rooms.get(code);
