1 bash · 1 read

The issue is likely mod_proxy_wstunnel — it's not mentioned in CLAUDE.md but it's required for WebSocket proxying. Also, the WebSocket path should have its own ws:// ProxyPass rule listed before the general one:

# WebSocket — must come first (more specific path)
ProxyPass /shift-tac-toe/ws ws://localhost:5780/ws
ProxyPassReverse /shift-tac-toe/ws ws://localhost:5780/ws

# Everything else
ProxyPass /shift-tac-toe/ http://localhost:5780/
ProxyPassReverse /shift-tac-toe/ http://localhost:5780/

And enable the module if you haven't:

sudo a2enmod proxy_wstunnel
sudo systemctl restart apache2

The upgrade=websocket param on a plain http:// ProxyPass rule isn't reliable — the separate ws:// rule with proxy_wstunnel is the correct approach.
