Assistant2026-03-14T01:20:12.438Z

Yeah, it's totally fine and actually the standard pattern — the inner nginx is just a lightweight static file server (~25MB alpine image), and Apache doesn't care what's behind the proxy. You'd just configure Apache something like:

<VirtualHost *:443>
    ServerName shift-tac-toe.yourdomain.com
    ProxyPass / http://localhost:8080/
    ProxyPassReverse / http://localhost:8080/
</VirtualHost>

The alternative would be ditching nginx and using vite preview or a Node static server, but those are worse choices — nginx is faster, handles caching headers properly, and the image stays small. Double-proxying adds essentially zero overhead for a game with a handful of concurrent players.
